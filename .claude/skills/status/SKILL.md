# Status コマンド

現在のセッションのコンテキスト使用状況を報告します。

## 概要

VSCode拡張ではCLIのステータスラインが表示されないため、このコマンドでコンテキスト状況を確認できます。

## 出力内容

このコマンドを実行すると、以下の情報を報告します：

### 1. コンテキスト使用状況

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 セッションステータス
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

コンテキスト使用率: {現在の使用率}%
[████████████░░░░░░░░]

状態: {良好 / 注意 / 警告 / 危険}
```

### 2. 状態の判定基準

| 使用率 | 状態 | 推奨アクション |
|--------|------|---------------|
| 0-50% | 良好 | そのまま作業継続 |
| 51-70% | 注意 | 大きなファイル読み込みに注意 |
| 71-85% | 警告 | 必要に応じて `/clear` を検討 |
| 86-100% | 危険 | `/clear` で新しいセッションを推奨 |

### 3. 追加情報

- 現在作業中のDD（あれば）
- 開発フローの現在ステップ
- 直近で読み込んだファイル数

## 使用タイミング

以下の場合に `/status` を実行してください：

1. **長時間作業時** - 定期的にコンテキスト状況を確認
2. **大きなファイルを読み込む前** - 余裕があるか確認
3. **レスポンスが遅くなったと感じた時** - コンテキスト圧迫の可能性
4. **複数のDDを参照した後** - 蓄積状況の確認

## 注意事項

- このコマンドはVSCode拡張環境向けの代替手段です
- CLI環境では `statusLine` 設定で常時監視が可能です（下記参照）

## CLI向けステータスライン設定（参考）

CLI環境で常時コンテキスト監視を行いたい場合は、以下の設定を参照してください：

### 設定手順

1. `~/.claude/statusline.js` を作成
2. `~/.claude/settings.json` に statusLine 設定を追加

詳細は [doc/cli-statusline-setup.md](../../../doc/cli-statusline-setup.md) を参照。

## 関連コマンド

| コマンド | 用途 |
|----------|------|
| `/clear` | セッションをクリアして新規開始 |
| `/dd status` | 開発フローの進捗確認 |
