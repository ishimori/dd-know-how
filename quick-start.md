# クイックスタート - 5分で始めるDD設計書

最小限の構成で今すぐDD設計書を使い始める手順です。

## Step 1: フォルダを作成（1分）

プロジェクトのルートに以下のフォルダを作成します。

```
your-project/
└── doc/
    ├── DD/           # 進行中のDD
    └── archived/
        └── DD/       # 完了したDD
```

## Step 2: テンプレートをコピー（1分）

[level-1-minimal/templates/dd_template.md](level-1-minimal/templates/dd_template.md) をプロジェクトにコピーします。

推奨の配置場所: `doc/templates/dd_template.md`

## Step 3: 最初のDDを作成（3分）

新しい機能やタスクを始めるとき、DDを作成します。

**ファイル名**: `DD-001_やりたいこと.md`

**例**: `DD-001_ログイン機能の実装.md`

テンプレートをコピーして、以下を埋めます：

```markdown
# DD-001: ログイン機能の実装

**ステータス**: 進行中

## 目的

ユーザーがID/パスワードでログインできるようにする。

## 背景・課題

- 現在は認証機能がない
- セキュリティ上、認証が必要

## 検討内容

- 認証方式: セッションベース vs JWT
- 決定: セッションベースを採用（シンプルなため）

## 決定事項

- セッションベース認証を採用
- パスワードはハッシュ化して保存

## タスク一覧

- [x] 認証サービスの実装
- [x] ログイン画面の作成
- [ ] テストの作成

## ログ

### 2024-01-15
- DD作成
- 認証方式を検討、セッションベースに決定

### 2024-01-16
- 認証サービス完了
- ログイン画面完了
```

## これだけで得られること

1. **何を決めたか記録される** - 後から「なぜこうした？」がわかる
2. **タスクが可視化される** - 何が終わって何が残っているか一目瞭然
3. **判断の経緯が残る** - ログセクションで意思決定を追跡

## 次のステップ

基本に慣れたら、以下にステップアップ：

- **Level 2**: スラッシュコマンド `/dd` で操作を効率化
- **Level 3**: 仕様書との連携で整合性を担保

詳細は [README.md](README.md) を参照してください。

---

## タスクステータスの記号

| 記号 | 意味 |
|------|------|
| `[ ]` | 未着手 |
| `[x]` | 完了 |
| `[-]` | スキップ/中止 |

## DD完了時

全タスクが完了したら：

1. ステータスを「完了」に変更
2. ファイルを `doc/archived/DD/` に移動

```
doc/DD/DD-001_ログイン機能.md
    ↓
doc/archived/DD/DD-001_ログイン機能.md
```

これで意思決定の履歴として保存されます。
